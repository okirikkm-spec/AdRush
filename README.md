# AdrenRush — рейтинг энергетиков Adrenaline Rush

Сайт для оценки вариантов энергетика Adrenaline Rush. Стек повторяет проект
TeachView: **Spring Boot (Java 21) + React + PostgreSQL + MinIO**, аутентификация
по JWT.

## Возможности

- **Главная** — карточки всех энергетиков в столбик, отсортированные по оценке.
- **Карточка-превью** — по клику всплывает описание и кнопка «Подробнее».
- **Страница энергетика** — большая фотография + галерея миниатюр (листаются,
  клик меняет главное фото), описание, оценки, форма отзыва (с правкой и
  удалением), загрузка собственного фото в конец галереи.
- **Аккаунты** — только логин + пароль (без почты). Скрыто сохраняется IP
  регистрации и последний IP — для будущей модерации мульти-аккаунтов.
- **Профиль** — аватарка, отображаемое имя, смена пароля, двухфакторная
  аутентификация (TOTP / Google Authenticator) с восстановлением пароля по коду,
  закрытый профиль, список своих отзывов и авто-тир-лист по оценкам.
- **Роли** — `USER` и `ADMIN`. Админ добавляет карточки и запускает парсер.
- **Парсер** — раз в сутки забирает последние записи с
  [drinks-energy.ru](https://drinks-energy.ru/category/adrenaline-rush/) и заводит
  карточки для новых напитков. При пустой базе делает полный первичный обход.

## Запуск через Docker (рекомендуется)

```bash
cp .env.example .env      # при необходимости поправьте значения
docker compose up --build
```

Откройте <http://localhost:8080>. Поднимутся PostgreSQL, MinIO и приложение
(React собирается внутрь Spring Boot и отдаётся со static).

Стартовый администратор: логин `admin`, пароль `admin123` (задаётся через
`ADMIN_USERNAME` / `ADMIN_PASSWORD`). **Смените пароль после первого входа.**

## Локальный запуск без Docker

Backend (профиль `local` — БД H2 в памяти, файлы в `uploads/`, MinIO не нужен):

```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=local
```

Frontend (dev-сервер на :3000 с проксированием /api на :8080):

```bash
cd src/frontend
npm install --legacy-peer-deps
npm start
```

## Технологии

| Слой      | Технологии                                                        |
|-----------|-------------------------------------------------------------------|
| Backend   | Spring Boot 3.4, Spring Data JPA, Spring Security, JWT (jjwt)      |
| 2FA       | TOTP (RFC 6238) собственной реализацией + ZXing для QR            |
| Парсинг   | Jsoup                                                              |
| Хранилище | PostgreSQL, MinIO (S3-совместимое) или локальная папка `uploads/` |
| Frontend  | React 18, React Router 6, Create React App                        |

## Основные API-эндпоинты

| Метод | Путь                               | Доступ        |
|-------|------------------------------------|---------------|
| POST  | `/api/auth/register` · `/login` · `/recover` | публично |
| GET   | `/api/drinks`                      | публично      |
| GET   | `/api/drinks/{id}`                 | публично      |
| POST  | `/api/drinks`                      | админ         |
| POST  | `/api/drinks/{id}/photos`          | пользователь  |
| POST  | `/api/drinks/parse?full=`          | админ         |
| GET/POST | `/api/drinks/{id}/reviews`      | чтение публично / запись авторизованно |
| GET   | `/api/users/{id}`                  | публично (учитывает приватность) |
| POST  | `/api/auth/2fa/setup` · `/enable` · `/disable` | авторизованно |
